Add 10 new secondary market instrument types, an AI document automation suite, and a regulatory document checklist system to the Aprisys deals platform. All changes are to the backend and frontend only — no database schema changes beyond adding new deal_type values and new document_type values.

PART 1 — NEW DEAL TYPES
Update the deal_type CHECK constraint in the database to add these new values. Run an ALTER TABLE statement:
sqlALTER TABLE cdi.deals DROP CONSTRAINT IF EXISTS deals_deal_type_check;
ALTER TABLE cdi.deals ADD CONSTRAINT deals_deal_type_check 
CHECK (deal_type IN (
  'equity','tbill','bond','private_placement','off_market','fixed_income',
  'commercial_paper','bankers_acceptance','promissory_note','reit_units',
  'eurobond','rights_nil_paid','agri_commodity_forward','sukuk',
  'infrastructure_bond','esos'
));
Also add new document types:
sqlALTER TABLE cdi.deal_documents DROP CONSTRAINT IF EXISTS deal_documents_document_type_check;
ALTER TABLE cdi.deal_documents ADD CONSTRAINT deal_documents_document_type_check
CHECK (document_type IN (
  'term_sheet','agreement','id_document','board_resolution','other',
  'letter_of_offer','subscription_agreement','deed_of_assignment',
  'comfort_letter','information_memorandum','programme_memorandum',
  'dealer_agreement','credit_rating','regulatory_filing','cscs_instruction',
  'aml_declaration','source_of_funds','pep_declaration','beneficial_ownership'
));
Also add a deal_metadata JSONB column to store instrument-specific fields:
sqlALTER TABLE cdi.deals ADD COLUMN IF NOT EXISTS deal_metadata JSONB DEFAULT '{}';

PART 5 — AI DOCUMENT GENERATION
Add 4 new endpoints to the AI router in backend/src/routes/other.ts:
POST /api/ai/generate-letter-of-offer
Input: { deal_id }
System prompt: You are a Nigerian capital markets legal expert. Generate a formal Letter of Offer for this OTC transaction. Include: date, parties (full names, addresses, RC numbers), subject line, instrument details, offer price, settlement terms (T+3 standard), conditions precedent, acceptance deadline, governing law (Laws of the Federation of Nigeria), jurisdiction (Lagos State High Court), signature blocks for both parties. Use professional legal language appropriate for Nigerian capital markets. Return plain text with clear section headers.
POST /api/ai/generate-subscription-agreement
Input: { deal_id }
System prompt: You are a Nigerian capital markets legal expert. Generate a Subscription Agreement for this OTC investment transaction. Include: recitals, definitions, subscription commitment, payment terms, representations and warranties from both parties, conditions to closing, investor eligibility confirmation, allotment terms, settlement mechanics, transfer restrictions, governing law (Laws of the Federation of Nigeria), dispute resolution (Lagos Court of Arbitration), execution block. Reference ISA 2025 and SEC Nigeria Rules 2024 where appropriate.
POST /api/ai/generate-deed-of-assignment
Input: { deal_id }
System prompt: You are a Nigerian capital markets legal expert. Generate a Deed of Assignment for the transfer of securities/instruments in this OTC transaction. Include: parties, recitals, definitions, assignment clause (absolute assignment), consideration, completion mechanics, seller's warranties (title, no encumbrances, authority), buyer's warranties, conditions, CSCS transfer instruction reference, stamp duty acknowledgment, governing law (Laws of the Federation of Nigeria), execution as a deed (requires witness signatures). This document transfers legal and beneficial ownership. Be precise and comprehensive.
POST /api/ai/generate-comfort-letter
Input: { deal_id, requesting_party_id }
System prompt: You are a Nigerian capital markets legal expert. Generate a Comfort Letter Request template that the buyer/investor can send to their bank to confirm fund availability. Include: date, addressee (the counterparty/seller), from (the buyer's bank name placeholder), re: deal reference, confirmation of available funds, account details placeholder, validity period (typically 5 business days), authorised signatory block for the bank. Include clear placeholders [BANK NAME], [ACCOUNT NUMBER], [AVAILABLE BALANCE] etc. that the party fills in before sending to their bank.
All 4 endpoints should:

Fetch deal + parties from DB
Call Claude claude-sonnet-4-5
Save generated document text to deal's ai_term_sheet equivalent — add a new ai_documents JSONB column to deals table:

sql  ALTER TABLE cdi.deals ADD COLUMN IF NOT EXISTS ai_documents JSONB DEFAULT '{}';

Store as { letter_of_offer: text, subscription_agreement: text, deed_of_assignment: text, comfort_letter: text }
Return the document text
Log to audit as ai.document_generated
